状态压缩

状态压缩DP，简称状压DP。

那状态压缩是什么捏？比如我们有一个vis数组，这个数组的第iii个元素代表这个元素选或不选。

那比如编号1,4,51,4,51,4,5的元素选了，0,2,30,2,30,2,3没有选的vis数组就可以这样表示：

这里vis数组的一系列权值都是0和1。只有0和1有没有让你想起什么？没错，2进制啊！！！

那如果想用一个二进制数表示这个vis数组，就可以这样：

那我们表示这个数组的对应的二进制数就是(010011)2(010011)_{2}(010011)2​。第iii个元素就对应了这个二进制数的从右往左第iii位。

至于为什么要从右往左，等你写代码你就会感到方便了。

那现在你对状态压缩是不是有点感觉了？状态压缩就是用一个二进制数来表示当前的状态，因为用数来表示操作会更快捷。

那长度为NNN的一个二进制数就可以表示NNN个元素的状态，范围是从000到2n−12^n-12n−1，所以在使用状压之前，要看看数据范围能不能容许你数达到2n−12^n-12n−1这个大小。

例题

不管怎样，先来几道状压DP的题练练手吧

旅行商

题目

题目描述

beat放假了想要去旅游，他准备去NNN个城市玩，这些城市分别编号111~NNN。

beat从111号城市开始，想玩遍所有NNN个城市，最后回到111号城市。

每个城市之间都可以坐飞机来往，从iii号城市到jjj号城市(1≤i,j≤n)(1 \leq i,j \leq n)(1≤i,j≤n)的机票是Ai,jA_{i,j}Ai,j​元。Ai,jA_{i,j}Ai,j​不一定等于Aj,iA_{j,i}Aj,i​。

beat不想重复玩同一个城市，请帮beat规划一种方案，使他花费的钱最少，并给出最少的钱数！

数据范围

对于100100100%的数据，1≤N≤181 \leq N \leq 181≤N≤18，如果i=ji = ji=j，Ai,j=0A_{i,j} = 0Ai,j​=0， 否则1≤Ai,j≤1041 \leq A_{i,j} \leq 10^41≤Ai,j​≤104。

时间限制1秒。

样例输入

样例输出

思路

对于每个城市，我们只有两种状态：去或者没去。分别可以用000和111表示。

注意到NNN最大到181818，非常的小，2n−12^n-12n−1最大到262144262144262144，然后我们还要枚举最后一次到达的城市是O(N)O(N)O(N)，那O(2NN)O(2^N N)O(2NN)正好能过1秒的时间限制。

我们先考虑状态，dpi,jdp_{i,j}dpi,j​就表示当前走过的城市状态iii，最后一次到达的城市jjj。

然后考虑转移：

我们第一维枚举去过的城市状态iii，i∈[1,2n−1]i \in [1,2^{n}-1]i∈[1,2n−1]。

我们第二维和第三维枚举最后一个去过的城市jjj和下一个要去的城市kkk，j,k∈[1,n]j,k \in [1,n]j,k∈[1,n]

现在考虑转移，首先我们要判断两个条件，看看现在的i,j,ki,j,ki,j,k能不能存在：

  1. dpi,jdp_{i,j}dpi,j​要存在，你连这个状态都不存在你怎么走到kkk呢？我们会把dpdpdp里的元素全部都变成无穷大，所以只需要判断dpi,jdp_{i,j}dpi,j​是否小于无穷大就行了。

  2. 题目说每一个城市不能重复去，所以我们要判断当前状态iii中，城市kkk有没有出现，那就是i >> (k - 1) & 1为000不就行了。那状态iii变成去过城市kkk就是iii的第kkk位要变成111，即新状态为i + (1 << (k - 1))。

那不就可以写出转移方程：

dp[i + (1 << (k - 1))][k] = min(dp[i + (1 << (k - 1))][k], dp[i][j] + a[j][k])

然后我们就做完了！

思路

首先对于每个角色池，只有可能抽过和没抽过两种状态，所以又可以用状压dp了，并不难

定义状态：dpidp_idpi​就表示当前状态iii的最优方案。

那我们第一维先枚举状态iii，然后第二维枚举选择的第一个池jjj，这里依旧要判定iii的第jjj位不能为111。

那这里的状态更新：

dp[i + (1 << (j - 1))] = min(dp[i + (1 << (j - 1))], dp[i] + a[j]);

题目里也可以同时抽两个池，所以要多一维kkk，这里kkk不能等于jjj，iii的第kkk位也不能是111。

题目描述

但是学习也很重要虽然没有看魔女之旅重要，beat在连续的MMM天里，最多只能有一半的天数看魔女之旅，请帮他规划哪几天可以追番，然后输出他最大可以获得的快乐值！

思路

这个MMM这么小才能到888，所以我们可以考虑从MMM入手

不难想到，可以定义dpi,jdp_{i,j}dpi,j​为第iii天时，前MMM天的状态为jjj能获得的最大快乐值就可以了

这个空间显然可以使用滚动数组进行优化，因为当前的iii只和i−1i-1i−1有关，可以考虑使用两个dp数组分别存储这一天的和前一天的状态即可，dpidp_idpi​就表示那一天是iii状态下可以获得的最大快乐值

但是我们还有一个问题，怎么快速判断这个状态是否存在呢？
